<threat_model>
## Primary Threat: Accidentally Placing Live Orders

**Attack surface:** The --live flag routes to live_run.run() which calls place_fak_order() against the real CLOB. If the container restarts with --live in any command override, real orders are placed.

**Mitigations this plan implements:**
1. Explicit command: ["python", "-m", "bot.main"] in docker-compose.yml — omits --live, making dry-run the default for all future `docker compose up -d` invocations.
2. Human-verify checkpoint confirms the running container CMD does NOT contain --live before this plan closes.

**What must NOT happen:**
- The command: override must NOT include --live.
- The restart command on VPS must NOT include --live.

**Residual risk:** A developer could still manually run `docker compose run bot --live` or `docker run ... python -m bot.main --live`. Mitigation: the command: override in docker-compose.yml is the canonical way to start the bot; the RESEARCH.md notes this pitfall explicitly.
</threat_model>
