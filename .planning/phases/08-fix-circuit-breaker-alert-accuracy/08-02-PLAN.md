<objective>
Fix Bug 1 (RISK-03): `engine.py` never calls `risk_gate.record_order_error()` when the NO leg exhausts all 3 retries. The hedge SELL runs but the circuit breaker is never notified of the failure — meaning repeated NO-leg failures cannot trip the CB. The fix adds a 2-line `hasattr` guard call at the top of the `if not no_filled:` block, mirroring the identical pattern already used for the YES verification failure path at line 327.

Purpose: NO-leg exhaustion is an order error (D-07: "order rejection, timeout, auth failure") and must be counted toward the CB sliding window so the breaker can trip on repeated failures.
Output: engine.py with record_order_error() call in NO exhaustion path; test_execution_engine.py with two new tests covering RISK-03.
</objective>
