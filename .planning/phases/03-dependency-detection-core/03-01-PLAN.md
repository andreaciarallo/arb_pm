    **_preprocess (DEP-01, per D-04 and D-05):**
    - `tokens = re.findall(r'\w+', question.lower())`
    - Filter out tokens in `_STOPWORDS` frozenset
    - Return `frozenset(filtered_tokens)`
    - `_STOPWORDS` is a module-level frozenset of ~35-40 words: "the", "a", "an", "is", "are", "was", "were", "will", "be", "been", "do", "does", "did", "have", "has", "had", "of", "in", "to", "for", "on", "at", "by", "from", "with", "as", "or", "and", "but", "if", "it", "its", "this", "that", "than", "any", "more", "not", "no"
    - CRITICAL (Pitfall 3): Stopwords include "by", "in", "more", "than" but signal functions DEP-03/04/05 operate on the ORIGINAL question string, NOT preprocessed tokens. Only Jaccard (DEP-02) uses preprocessed tokens.

    **_jaccard_similarity (DEP-02, per D-08):**
    - `len(tokens_a & tokens_b) / len(tokens_a | tokens_b)`
    - Return 0.0 if either set is empty

    **_keyword_implication (DEP-03, per D-09):**
    - Operates on ORIGINAL question strings (not preprocessed tokens)
    - Curated implication rules as list of `(child_pattern, parent_pattern)` regex tuples:
      - `(r'win\s+by\s+\d', r'\bwin\b')` -- "win by X%" implies "win"
      - `(r'beat\s+.*\s+by\s+\d', r'\bbeat\b')` -- "beat Z by X" implies "beat Z"
      - `(r'reach\s+\$[\d,]+[kKmMbB]?', r'\breach\b')` -- "reach $X" implies "reach"
      - `(r'pass\s+\$[\d,]+[kKmMbB]?', r'\bpass\b')` -- "pass $X" implies "pass"
      - `(r'more\s+than\s+\d', r'\bmore\b')` -- "more than X" implies concept
    - For each rule: if question_a matches child_pattern AND question_b matches parent_pattern (or vice versa), return 1.0
    - If no rule matches, return 0.0
    - Check both directions (a->b and b->a)

    **_numeric_relation (DEP-04, per D-10):**
    - Operates on ORIGINAL question strings
    - Extract numbers using regex: percentages `r'(\d+\.?\d*)%'`, dollar amounts `r'\$(\d[\d,]*\.?\d*)[kKmMbB]?'`, plain numbers `r'\b(\d+\.?\d+)\b'`
    - Normalize dollar suffixes: k=1000, m/M=1000000, b/B=1000000000
    - If both questions have exactly one extracted number: check if one is strictly greater (containment). If so, return 1.0. If equal or no numbers, return 0.0
    - If different count of numbers, return 0.0

    **_time_relation (DEP-05, per D-11):**
    - Operates on ORIGINAL question strings
    - Extract dates using regex handling all 4 Polymarket formats:
      - `"in 2025"` -> (2025, 12, 31) -- treat year-only as end of year
      - `"by December 31"` -> (current_year, 12, 31) -- no year = current year
      - `"by March 31, 2026"` -> (2026, 3, 31)
      - `"by June 30, 2026?"` -> (2026, 6, 30)
    - Use `calendar.month_name` for month mapping (lowercase keys)
    - Compare extracted dates: if question_a's date is strictly EARLIER than question_b's date, return 1.0 (earlier deadline is subset of later). If b < a, also return 1.0 (bidirectional, since we detect relationship not direction).
    - Same date -> return 0.0 (not a subset relationship)
    - No dates extracted -> return 0.0

    **_event_bonus (DEP-06, per D-12):**
    - If both event_id_a and event_id_b are non-None and equal, return 1.0
    - Otherwise return 0.0
