## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PriceCache -> Simulator | Cached price data (potentially stale or manipulated) enters P&L calculation |
| Market question text -> SQLite | User-supplied text from Polymarket API inserted into paper_trades table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | simulator.py P&L calculation | mitigate | Validate that vwap_yes > 0 and vwap_no > 0 before division; guard kelly_usd > 0 before computing kelly_fraction |
| T-05-02 | Tampering | schema.py insert_paper_trade | mitigate | Use parameterized queries (? placeholders) for all SQL inserts -- never string interpolation. Already established pattern in schema.py |
| T-05-03 | Information Disclosure | paper_trades table | accept | Paper trades contain simulated data only, no secrets or PII. Single-user bot with local SQLite |
| T-05-04 | Tampering | PriceCache data integrity | accept | PriceCache is populated by existing validated WebSocket/HTTP pipeline with math.isfinite() checks. Paper simulator inherits same protections |
</threat_model>
