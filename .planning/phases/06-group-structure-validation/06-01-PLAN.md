```python
"""
Group structure validation for Polymarket event groups (D-01).

Validates event groups as one-of-N partitions suitable for basket arbitrage.
NegRisk groups auto-pass (exchange-guaranteed). Non-NegRisk groups are checked
for structural violations: duplicates, subsets, overlaps.

Runs ONCE at startup (D-02). Detection loop checks membership via get_valid_groups().
"""
import json
import itertools
from collections import defaultdict

from loguru import logger

from bot.detection.cross_market import EventInfo, _event_groups, _gamma_market_data
from bot.detection.dependency import (
    _preprocess,
    _jaccard_similarity,
    _keyword_implication,
    _extract_number,
)
```

Module-level cache:
```python
_valid_groups: set[str] = set()
```

Constants:
```python
_DUPLICATE_THRESHOLD = 0.9   # Jaccard similarity for duplicate detection (GV-02)
_SUBSET_JACCARD_MIN = 0.6    # Minimum Jaccard for numeric threshold check (GV-03)
_COMPLETENESS_LOW = 0.7      # Mid-price sum lower bound (GV-04)
_COMPLETENESS_HIGH = 1.3     # Mid-price sum upper bound (GV-04)
```

Functions to implement:

1. `is_duplicate_pair(question_a: str, question_b: str) -> tuple[bool, float]`:
   - Use `_preprocess()` + `_jaccard_similarity()` from dependency.py
   - Return `(score > _DUPLICATE_THRESHOLD, score)`

2. `is_subset_pair(question_a: str, question_b: str) -> tuple[bool, str]`:
   - Signal 1: `_keyword_implication(question_a, question_b) > 0.0` -> return `(True, "keyword_implication")`
   - Signal 2: If `_jaccard_similarity(tokens_a, tokens_b) > _SUBSET_JACCARD_MIN`, extract numbers with `_extract_number()`. If both have numbers and numbers differ -> return `(True, "numeric_threshold")`
   - Default: return `(False, "")`

3. `passes_completeness_check(markets: list[dict], low: float = _COMPLETENESS_LOW, high: float = _COMPLETENESS_HIGH) -> tuple[bool, float]`:
   - Sum YES prices from `outcomePrices` field (index 0 of JSON-parsed list)
   - Parse with `json.loads()` if string, handle list directly if already parsed
   - Return `(low <= mid_sum <= high, mid_sum)`

4. `_validate_non_negrisk_group(event_id: str, cids: list[str]) -> bool`:
   - Get questions from `_gamma_market_data` for each cid
   - Check all pairs with `itertools.combinations()`:
     a. `is_duplicate_pair()` -> if True, log `GV-REJECT: duplicate | q1="..." q2="..." | score=X.XX` and return False
     b. `is_subset_pair()` -> if True, log `GV-REJECT: subset | q1="..." q2="..." | signal=keyword_implication` and return False
   - Get market data from `_gamma_market_data` for completeness check
   - Build market dicts with outcomePrices for `passes_completeness_check()`
   - If completeness fails -> log `GV-REJECT: completeness | event={event_id} | mid_sum=X.XX` and return False
   - Return True (group is valid)

5. `validate_groups() -> set[str]`:
   - Build event_id -> list[condition_id] mapping from `_event_groups`
   - For each event_id:
     - Get EventInfo (from any cid in the group)
     - If `info.neg_risk` is True: add to valid set, increment neg_risk_count
     - Else: call `_validate_non_negrisk_group(event_id, cids)`, add to valid if True
   - Log aggregate at INFO level per D-10: `"GV: {neg_risk_count} NegRisk groups auto-validated, {checked_count} non-NegRisk checked, {rejected_count} rejected"`
   - Update `_valid_groups` global and return the set

6. `get_valid_groups() -> set[str]`:
   - Return `_valid_groups` (module-level cache accessor)
