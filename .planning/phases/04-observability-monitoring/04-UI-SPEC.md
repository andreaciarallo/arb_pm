# Phase 4 — UI Design Contract

> Visual and interaction contract for the FastAPI monitoring dashboard. Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

This phase delivers a **single-page, internal ops dashboard** served as inline HTML/CSS/JS via FastAPI `HTMLResponse`. No component library, no CDN, no external assets. Everything must work offline on the VPS. Data legibility is the primary design value over aesthetics.

## Typography

All fonts are system stack — no Google Fonts, no CDN. VPS offline operation is guaranteed.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 13px | 400 | 1.4 | system-ui, -apple-system, "Segoe UI", sans-serif | Default text, table cell text, status descriptions |
| Label | 11px | 600 | 1.2 | system-ui, -apple-system, "Segoe UI", sans-serif | Column headers, section labels, badge text, metric unit suffixes |
| Heading | 16px | 600 | 1.2 | system-ui, -apple-system, "Segoe UI", sans-serif | Section headings (Status, Trades, Analytics), page title |
| Display | 24px | 600 | 1.0 | system-ui, -apple-system, "Segoe UI", sans-serif | Daily P&L figure, capital balance (the single most important number on the page) |
| Mono | 13px | 400 | 1.4 | "SF Mono", "Fira Code", "Cascadia Code", Consolas, monospace | All numeric data: prices, P&L values, sizes, timestamps, arb IDs |

**Typography rules:**
- All monetary values use monospace at 13px weight 400.
- Positive P&L values render in `#22c55e` (profit green). Negative P&L values render in `#f87171` (loss red). Zero renders in the secondary text color.
- Timestamps display as `HH:MM:SS UTC` in monospace 11px, label weight 600.
- Market question text truncates at 48 characters with `text-overflow: ellipsis` — dashboard density requires this.

## Component Inventory

This phase uses no component library. The following HTML/CSS primitives make up the entire dashboard:

### Status Bar (top of page)

Single horizontal strip. Background: `--bg-card`. Padding: `md` (16px). Content: bot status dot + label, circuit breaker state, kill switch state, cycle count, last scan time.

**Bot status dot:** 10px circle, color matches status color variable. Positioned inline-left of status text.

Layout: `display: flex; justify-content: space-between; align-items: center;`

### Stat Card Grid

6 cards in a 3-column grid (2 rows of 3). Each card:
- Background: `--bg-card`
- Border: `1px solid --border`
- Padding: `md` (16px)
- Label: 11px, weight 600, `--text-secondary`
- Value: 24px (Daily P&L card) or 16px (other cards), weight 600, `--text-primary` (with P&L color override)
- Border-radius: 6px

Cards in order: Daily P&L | Total Capital | Open Positions | Total Trades | 7-Day Efficiency | 30-Day Efficiency

### Trades Table

Full-width. Columns (left to right):

| Column | Width | Alignment | Format |
|--------|-------|-----------|--------|
| Time | 72px | center | `HH:MM:SS` mono 11px |
| Market | flex (min 200px) | left | truncated at 48 chars, 13px |
| Leg | 40px | center | `YES` / `NO` label 11px bold |
| Size (USD) | 72px | right | `$0.00` mono 13px |
| Price | 60px | right | `0.0000` mono 13px |
| Fees (USD) | 64px | right | `$0.0000` mono 13px |
| Net P&L | 72px | right | `+$0.00` / `-$0.00` mono 13px, colored |
| Status | 76px | center | badge (see below) |

Table header: `--bg-card`, label 11px weight 600, `--text-secondary`, `position: sticky; top: 0`.

Row hover: `background: #1e2130` (4% lighter than `--bg-row-alt`).

### Status Badge

Inline-block, border-radius 4px, padding `4px 8px` (xs + sm). Font: 11px weight 600. No border (background-only badge).

## Copywriting Contract

### Bot Status Labels

| State | Label | Description shown below |
|-------|-------|------------------------|
| running | RUNNING | Scan cycle active |
| paused | PAUSED | Stop-loss limit reached — trading halted until midnight UTC |
| blocked | BLOCKED | Circuit breaker open — cooldown Xm Xs remaining |
| stopped | STOPPED | Kill switch active — all positions closing |
| degraded | DEGRADED | Bot process unreachable — check VPS |

**Display format:** Status dot + label in matching color, description in `--text-secondary` body 13px beneath.

### P&L Display Format

| Value | Display | Color |
|-------|---------|-------|
| Positive | `+$12.3456` | `#22c55e` |
| Negative | `-$3.2100` | `#f87171` |
| Zero | `$0.0000` | `--text-secondary` |
| Unavailable | `—` | `--text-secondary` |

All P&L values show 4 decimal places (USDC precision). Daily P&L display card shows 2 decimal places for readability.

### Empty States

| Section | Heading | Body |
|---------|---------|------|
| Trades table (no trades yet) | No trades recorded | Bot is scanning. Executed trades will appear here. |
| Arb analytics (no completed arbs) | No completed arbs | Arb pairs appear here after both legs are confirmed filled. |
| Open positions (none) | No open positions | — (omit body; show only in the stat card as `0`) |
| Capital efficiency (insufficient data) | N/A | Need at least one completed arb to compute efficiency. |

Empty state styling: centered within the table body, `--text-secondary`, body 13px, no icon.

### Error States

| Error | Message | Recovery hint |
|-------|---------|---------------|
| API fetch failed (JS) | Data refresh failed | Retrying in 10s — check bot logs if this persists |
| Dashboard endpoint 500 | Dashboard error | Bot process may be unhealthy — check `docker compose logs bot` |
| No data returned | No data | Bot may have just started — refreshing in 10s |

Error state styling: top of page below status bar, `background: rgba(248,113,113,0.10)`, `border: 1px solid rgba(248,113,113,0.3)`, `color: #f87171`, padding `sm` (8px), border-radius 4px. Dismisses automatically on next successful fetch.

### Circuit Breaker Messages

| State | Dashboard label | Telegram alert text |
|-------|----------------|---------------------|
| Trip | BLOCKED — circuit breaker open | Circuit breaker tripped. Errors: {N}/60s. Cooldown: {X}m {Y}s. |
| Cooling (in countdown) | BLOCKED — cooldown {Xm Ys} remaining | (no additional alert during cooldown) |
| Reset | RUNNING | (no alert on reset — silence is signal that bot recovered) |

### Kill Switch Messages

| Trigger | Dashboard label | Telegram alert text |
|---------|----------------|---------------------|
| KILL file detected | STOPPED — kill switch active (file) | Kill switch triggered via KILL file. Closing positions now. |
| SIGTERM received | STOPPED — kill switch active (signal) | Kill switch triggered via SIGTERM. Closing positions now. |

### Daily Summary Telegram Format

```
Daily Summary — {YYYY-MM-DD UTC}

P&L: +$X.XXXX
Trades: N (W wins, L losses, win rate X%)
Arbs completed: N
Fees paid: $X.XXXX
Capital efficiency (today): +X.XX%

Bot status: RUNNING
```

Plain text (no HTML parse mode for daily summary — reduces escaping risk). Each item on its own line.

### Completed Arb Telegram Format

```
Arb complete — {market question truncated 60 chars}

YES: {yes_entry_price:.4f} | NO: {no_entry_price:.4f}
Size: ${size_usd:.2f} | Hold: {hold_time}
Gross: +${gross_pnl:.4f} | Fees: ${fees_usd:.4f}
Net P&L: +${net_pnl:.4f}
```

Use HTML parse mode (`parse_mode="HTML"`). Market question wrapped in `<b>` tags only. All numeric values plaintext (no bold) — avoids HTML entity escaping issues on dynamic content.

### Destructive Actions

No destructive actions are exposed in the dashboard UI in Phase 4. The dashboard is read-only. Kill switch is activated via filesystem (`/app/data/KILL` file) or SIGTERM — not via the dashboard UI. No confirmation dialogs required.

## Interaction Contract

### Auto-Refresh (JS setInterval)

- `setInterval(refresh, 10000)` — every 10 seconds
- `refresh()` called immediately on page load (no blank state on first render)
- Countdown timer updates every 1 second via a separate `setInterval(countdown, 1000)`
- On fetch error: show error banner, do NOT clear existing data — stale data is better than blank
- On fetch success after error: dismiss error banner, update all values

### No User-Initiated Actions

The dashboard is fully read-only in Phase 4. No buttons, no forms, no inputs. The only interaction is browser scroll for long tables. No confirmation modals, no tooltips, no hover-expand panels.

### Data Staleness Indicator

If `setInterval` fetch fails 3+ consecutive times: append `"(stale — last update HH:MM:SS UTC)"` to the status bar in `--status-blocked` color (`#f87171`). Normal operation shows no staleness indicator.

## Implementation Notes for Executor

1. **Single HTML string.** The entire dashboard is one `HTMLResponse` returned from the `/` route. Inline `<style>` block at the top of `<head>`. Inline `<script>` block at the bottom of `<body>`. No external files.

2. **CSS variables.** Declare all color tokens as CSS custom properties on `:root`. This makes the contract auditable — every color reference in the HTML must map to a variable name from the Color table above.

3. **Monospace data alignment.** Use `font-variant-numeric: tabular-nums` on all numeric table cells to prevent column jitter on value change. This is a CSS property — no JS required.

4. **Table scroll.** Wrap the Trades table in a `<div style="overflow-x: auto">` to handle narrow viewports on VPS SSH tunnels (800px-wide terminal browser).

5. **No `innerHTML` for user data.** All dynamic data injected via `element.textContent = value` or `element.innerText = value` — never `innerHTML`. Market question text from SQLite is untrusted and must not be rendered as HTML.

6. **Timestamp format.** All timestamps displayed as `HH:MM:SS UTC`. ISO strings from the API are parsed with `new Date(isoString).toLocaleTimeString('en-GB', {timeZone: 'UTC', hour12: false})`.

7. **P&L sign prefix.** Positive values always display `+$X.XX` (explicit plus sign). Negative always `-$X.XX`. Zero displays `$0.00` (no sign). Implement as: `const sign = v > 0 ? '+' : ''; return sign + '$' + Math.abs(v).toFixed(4)`.
