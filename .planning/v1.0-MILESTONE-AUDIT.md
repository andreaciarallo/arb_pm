## Critical Gap: Token IDs Never Reach Execution Engine

**Severity:** BLOCKER — No live trades ever execute

**Root cause:** `ArbitrageOpportunity` dataclass (Phase 2) has no `yes_token_id` / `no_token_id` fields. The detection engines (`yes_no_arb.py`, `cross_market.py`) resolve token IDs as local variables for cache lookups but discard them before returning the opportunity. `live_run.py` calls `execute_opportunity(client, opp, config, risk_gate)` — no token ID args. `engine.py` Gate 0 checks `if not yes_token_id or not no_token_id` → always true → `ExecutionResult(status="skipped")` returned for every opportunity.

**Evidence chain:**
1. `src/bot/detection/opportunity.py` — `ArbitrageOpportunity` has no `yes_token_id` field
2. `src/bot/detection/yes_no_arb.py:60-67` — token IDs are local variables, not stored in returned opportunity
3. `src/bot/live_run.py:297` — `execute_opportunity(client, opp, config, risk_gate)` — no token ID args
4. `src/bot/execution/engine.py:144-162` — Gate 0: `if not yes_token_id or not no_token_id: return skipped`

**Impact:** Bot detects opportunities correctly, logs "skipped" records to SQLite, but places zero orders. All of EXEC-01 through EXEC-04 and RISK-01 are architecturally unreachable.

**Fix:** Add `yes_token_id: str` and `no_token_id: str` to `ArbitrageOpportunity` dataclass. Populate in `yes_no_arb.py` and `cross_market.py`. Update `live_run.py` call site (no new args needed — token IDs will be on `opp`).

## Minor Gap: Telegram Kill Switch and Circuit Breaker Alerts Not Wired

**Severity:** MINOR — OBS-02 partial

`TelegramAlerter.send_kill_switch()` and `send_circuit_breaker_trip()` are implemented but have zero call sites in the codebase. Users never receive Telegram notifications when the kill switch fires or the circuit breaker trips — two of the most critical bot events.

**Fix:** Add `asyncio.create_task(alerter.send_kill_switch(...))` to `_execute_kill_switch()` in `live_run.py`. Add `asyncio.create_task(alerter.send_circuit_breaker_trip(...))` when circuit breaker trips (inside scan loop after `risk_gate.record_order_error()`).
