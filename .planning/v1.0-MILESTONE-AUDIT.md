## Integration Gaps

### Gap 1 — RISK-03: NO Leg Failures Don't Trip Circuit Breaker

**File:** `src/bot/execution/engine.py` — NO retry loop (approx lines 369–415)

`risk_gate.record_order_error()` is called only when YES verification fails (`verify_fill_rest` returns False). The NO leg retry loop (3 retries × 500ms) does not call `record_order_error()` when all retries are exhausted. Systematic NO-leg failures will not cause the circuit breaker to trip.

**Fix:** Call `risk_gate.record_order_error()` after the NO retry loop exits with `not no_filled`.

---

### Gap 2 — OBS-02: CB Alert Shows Static Threshold, Not Live Count

**File:** `src/bot/live_run.py` line 399

`alerter.send_circuit_breaker_trip(error_count=risk_gate.circuit_breaker_errors, ...)` passes the *configured* CB threshold (default `5`) as `error_count`. The actual `_error_timestamps` list is cleared in `gate.py` at trip time, so the live count is lost before the alert fires. The alert always shows the static configured threshold.

**Fix:** Record the triggering count before clearing `_error_timestamps` in `RiskGate.record_order_error()` and expose it as a property.

---

### Observation — Cross-Market Execution Skipped by Design

`cross_market.py` sets `no_token_id=""` (D-01 Phase 05), causing Gate 0 to skip all cross-market opportunities. Skipped results get `insert_trade()`-d and inflate `app_state.total_trades`. No change needed — this is documented design — but worth noting for dashboard display.
