<objective>
Fix EXEC-SIZE-001: `place_fak_order` passes `size_usd` directly to `OrderArgs.size`, but the
Polymarket CLOB interprets `size` as number of outcome tokens. Total USDC spent = size_tokens × price,
so a $5 order at price 0.50 must send `size=10.0` tokens, not `size=5.0`.

Purpose: Prevent systematic under-sizing of every order — at price 0.50 the bot spends half the
intended capital on each trade.
Output: Corrected `order_client.py` with `size_tokens = size_usd / price` conversion + regression
test proving the conversion.
</objective>
