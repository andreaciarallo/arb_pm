<objective>
Fix Bug 2 (OBS-02): the circuit breaker Telegram alert currently passes `risk_gate.circuit_breaker_errors` (a static configured integer, e.g. 5) to `alerter.send_circuit_breaker_trip(error_count=...)`. After `record_order_error()` trips the CB it immediately clears `_error_timestamps`, so the live triggering count is lost. The fix captures `len(_error_timestamps)` into a new `_last_trip_count` instance attribute before the clear, exposes it as `last_trip_error_count` property on `RiskGate`, then replaces the one-word argument at the live_run.py call site.

Purpose: Alert messages must reflect what actually happened (7 errors burst) not a constant config value, so the on-call operator knows the actual severity of each CB trip.
Output: gate.py with new property, live_run.py with corrected argument, test_live_run.py with new OBS-02 test and updated regression guard.
</objective>
