<objective>
Add `yes_token_id` and `no_token_id` fields to the `ArbitrageOpportunity` dataclass and wire them through the detection engines so downstream consumers (engine.py Gate 0) receive non-empty token IDs.

Purpose: This is the first half of the TOKEN-ID-GAP fix. Without these fields populated on the dataclass, Gate 0 in engine.py receives empty strings on every opportunity and returns `status='skipped'` — making live trade execution unreachable.

Output:
- `opportunity.py` with two new string fields (defaults `""`)
- `yes_no_arb.py` populates both fields from already-resolved local vars
- `cross_market.py` populates `yes_token_id` from `group[0]`'s YES token; `no_token_id=""` (D-01)
- New test in `test_yes_no_arb.py` asserting token IDs are non-empty on returned opportunities
</objective>
