# Milestone v1.1: Cross-Market Detection & Execution Fixes

**Status:** ✅ SHIPPED 2026-04-19
**Phases:** 1 (post-v1.0 Phase 1)

## Overview

Single-phase milestone closing the three root causes identified after v1.0 shipped: cross-market false positives from BFS keyword heuristic, missing token ID wiring in the execution engine, and the bot running in `--live` mode without a dry-run gate. Also hardened the VPS against SSH brute-force attacks.

## Phases

### Phase 1: Fix Cross-Market False Positives & Wiring

**Goal:** Fix the three root causes: switch VPS bot to dry-run, replace keyword heuristic in cross-market detector with Gamma API event-level grouping, and wire cross-market execution with per-leg token IDs.
**Requirements:** DRY-RUN, CROSS-DETECT, CROSS-EXEC
**Depends on:** Phase 0 (v1.0 MVP)
**Plans:** 4 plans

Plans:

- [x] 01-01: Switch VPS to dry-run and lock via docker-compose.yml command override

- [x] 01-03: Add legs field to ArbitrageOpportunity and wire cross-market execution in engine.py

**Details:**

Plan 01-01 switched the VPS docker-compose.yml to use `command: [python, -m, bot.main]` (no `--live` flag), confirmed via logs. Also hardened the VPS: UFW enabled, fail2ban installed, PasswordAuthentication disabled after SSH brute-force attack detected.

Plan 01-02 replaced `_extract_keywords()` + `_group_markets()` (BFS connected-components) with `load_event_groups()` (Gamma API `conditionId→event_id`) + `_group_by_event()`. Confirmed CLOB market objects have no `event_id` field — PATH B (Gamma API) is the only viable approach. Fallback chain: Gamma → `neg_risk_market_id` → `neg_risk_id`. 8 tests passing.

Implemented `_execute_cross_market()` with equal-shares sizing (`target_shares = kelly_usd / total_yes`), per-leg FAK BUY orders, and partial-fill hedge (SELL at price=0.01). Gate 0 dual-routing: `opportunity_type == "cross_market" and opp.legs` → cross-market path; YES+NO unchanged. 13 tests passing.

Plan 01-04 imported `load_event_groups` in both runner files and called it after `fetch_liquid_markets`, before `WebSocketClient`. Outer try/except marked `# pragma: no cover` (inner function already swallows all exceptions). Closes UAT Test 4 gap. 2 new tests (TDD); 109 total unit tests passing.

## Milestone Summary

**Key Decisions:**

- PATH B (Gamma API) chosen for cross-market grouping — CLOB has no direct `event_id` on market objects; `conditionId` in Gamma = `condition_id` in CLOB
- Equal shares not equal dollars for cross-market sizing: `target_shares = kelly_usd / total_yes`
- Partial hedge sells ALL previously filled legs at price=0.01 on any leg failure
- Gate 0 dual-routing requires both `opportunity_type==cross_market AND legs` populated to prevent YES+NO bypass
- `load_event_groups()` called once at scanner startup (not in hot detection path); `_event_groups` is a module-level cache

**Issues Resolved:**

- Cross-market false positives from BFS keyword heuristic — eliminated by Gamma API event-level grouping
- Missing token IDs in execution engine — resolved by `legs` field on ArbitrageOpportunity
- VPS running in `--live` mode without safeguard — resolved by docker-compose.yml override
- VPS SSH brute-force attack (45.227.254.170) — resolved by UFW + fail2ban

**Issues Deferred:**

- `_event_groups` empty until Gamma API call succeeds at startup — fallback to `neg_risk_market_id` (covers NegRisk subset only)
- Cross-market execution still in dry-run only (all bot executions gated) — live cross-market trades pending post-v1.1 validation

**Technical Debt Incurred:**

- Phase 1 has no formal VERIFICATION.md — verified via TDD (109 tests) + UAT (8/8 passed) rather than GSD verification agent
- `pragma: no cover` on outer try/except in dry_run.py/live_run.py startup guard
